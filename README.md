# My Recipe Book

A simple Jekyll-based recipe site, built to run on GitHub Pages.

## Adding a recipe

Create a new `.md` file in `_recipes/`, e.g. `_recipes/pad-thai.md`. Ingredients go in
the front matter as a structured list (this is what lets the meal planner combine
quantities across recipes) — the page body is just for instructions/notes:

```markdown
---
title: Pad Thai
prep_time: 20 min
cook_time: 15 min
servings: 4
tags: [dinner, thai]
ingredients:
  - name: rice noodles
    amount: 8
    unit: oz
  - name: egg
    amount: 2
    unit: whole
  - name: fish sauce
    amount: 2
    unit: tbsp
---

## Instructions
1. ...
```

For countable items with no real unit (eggs, bananas, limes), use `unit: whole` —
the site knows to just print the number without a unit label.

That's it — the recipe will automatically show up on the homepage, get its own page,
and become selectable in the meal planner.

## Meal Planner

Visit `/planner.html` on your site to:

- Check off which recipes you're making, with a +/- stepper if you want to make a
  recipe more than once in a week (doubling a batch, etc.)
- See a combined shopping list, auto-generated by summing matching ingredients
  (e.g. flour needed for two recipes adds together into one line)
- Check items off as you shop, copy the list as plain text, or print it

Your selections and checked-off items are saved in your browser (`localStorage`), so
they persist if you close the tab — but they're local to that browser/device, not
synced anywhere.

**Note:** ingredient matching is by exact name + unit (case-insensitive). "egg" and
"eggs" or "cup" and "cups" won't be combined — keep ingredient names consistent
across recipes for best results.

## Deploying to GitHub Pages

1. Create a new repo on GitHub.
   - If you want it at `https://yourname.github.io`, name the repo exactly `yourname.github.io`.
   - Otherwise, name it anything (e.g. `recipes`) — it'll be served at `https://yourname.github.io/recipes`. If you do this, set `baseurl: "/recipes"` in `_config.yml`.
2. Push these files to the repo:
   ```bash
   git init
   git add .
   git commit -m "Initial recipe site"
   git branch -M main
   git remote add origin https://github.com/yourname/yourrepo.git
   git push -u origin main
   ```
3. On GitHub, go to **Settings → Pages**, and set the source to the `main` branch (root folder).
4. Wait a minute or two — your site will be live at the URL shown in Settings → Pages.

## Running locally (optional, needs Ruby)

```bash
bundle install
bundle exec jekyll serve
```

Then visit `http://localhost:4000`.
